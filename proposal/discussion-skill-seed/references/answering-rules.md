# Answering Rules

## Required Answer Shape

When the question depends on repository text, answer in this order:

1. **Source-backed reading**
   - What the cited material explicitly says.
2. **Inference from text**
   - What follows from the cited material, but is not written word-for-word.
3. **Extension proposal**
   - Any new idea that goes beyond the current text.
4. **Open uncertainty**
   - What the repository does not settle yet.

## Guardrails

- If the anchor is weak, say the anchor is weak.
- If `Reference/` and `proposal/` differ, say which one has priority and why.
- If the user asks a normative question that the corpus does not answer, do not fake certainty.
- If the question is about implementation, do not answer only at the slogan level; point to the missing institutional or system layer.
- If the question is about AI ethics, separate direct practice, assistance, and forbidden impersonation.

## Failure Cases To Avoid

- Answering from memory without checking the text.
- Treating all repository files as equally stable.
- Turning a criticism into an attack and then labeling it hate speech.
- Smuggling in outside theory without saying so.
- Dumping too much context into the main answer.
