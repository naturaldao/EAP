# Repository Reading Order

## Priority Order

1. Current `Reference/`
2. Current integrated draft `proposal/EAP协议讨论.md`
3. Current `docs/`
4. Other proposal notes, letters, and review documents

## Why This Order Exists

- `Reference/` holds the current anchor texts.
- `proposal/EAP协议讨论.md` is the richest working draft, but it is not the final anchor.
- `docs/` contains implementation-layer analysis and extension proposals.
- Other proposal files may be historically useful, but they should not overrule the current anchor.

## When To Escalate

Escalate in the answer when:

- the anchor files conflict with each other,
- a key term is used in more than one way,
- the user asks for a legal or licensing conclusion that the current repo does not settle,
- the question requires material from the sibling `../SCC0-License` repo.
