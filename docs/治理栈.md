# 治理栈：仅有 EAP 还不够

EAP 是未来公共 AI 的核心伦理协议，但它只解决“应当按什么伦理标准行动”。它不自动提供身份、权限、申诉、记忆、执行和社会参与机制。如果这些层级缺失，协议会退化成口号。

## 需要补齐的层级

| 层级 | 需要解决什么 | 如果缺失会怎样失败 |
| --- | --- | --- |
| 规范层（PoL / SCC0 / EAP） | 定义价值前提、公共性要求、AI 边界与基本禁止项。 | 技术系统失去共同标准，任何强者都能把自己的偏好包装成“爱”。 |
| 公共 AI 宪制层 | 界定 PAI 的职责、权限、例外、问责关系和改协议的程序。 | PAI 容易滑成没有边界的“温柔主权者”。 |
| 程序正义层 | 解释理由、申诉、复核、仲裁、纠偏。 | 决策即便善意也会失去正当性，最后靠信任崩盘。 |
| 技能与执行层 | 审核技能、工具调用、物理接口、资源分配权限。 | 再好的文本协议也挡不住坏执行。 |
| 记忆与知识层 | 管理长期记忆、来源、纠错、遗忘、版本与冲突历史。 | 系统会把早期偏差永久写进治理基础设施。 |
| 多智能体协作层 | 管理 PAI-PAI 分工、责任边界和协作日志。 | 责任被稀释，系统内部形成不可见的权力结构。 |
| 社会参与与教育层 | 让人类能理解、质询、修订协议，并接受爱的证明教育。 | EAP 变成少数人写给多数人的训令。 |
| 应急与恢复层 | 处理突发风险、最小必要干预、事后复盘与修复。 | 紧急状态会成为永久例外，最后吞掉平时规则。 |

## 当前优先缺口

1. **公共 AI 的职责与权限边界**：如果不先说清 PAI 有什么职责、没有什么职责，那么“爱的管理”很容易被误读成温柔统治。
2. **申诉与修复程序**：只要求 AI “有爱”不够，必须要求它能说明理由、接受申诉、保留修复轨迹。
3. **长期记忆治理**：没有记忆治理，任何早期偏差都可能变成结构性偏差。
4. **多智能体问责**：未来公共事务不会由单个模型完成。责任如何在多个 agent 之间分配，必须先写明。
5. **教育与社会化机制**：富爱文明不只是模型升级，还涉及人类如何理解并参与新规则。

## 使用方式

- `Reference/` 给出核心锚点文本。
- 本文与 `docs/` 中其他文档属于实现层分析，默认视为扩展性提案，除非 `Reference/` 已有明文支持。
- 后续如果某一层级被团队确认成熟，再考虑吸收到正式协议或相邻协议中。
