# 系统架构：如果未来社会真的让 PAI 治理公共事务，需要什么系统

EAP 不是一段 prompt。它需要落在一套可解释、可审计、可控权的系统里。

## 核心组件

| 组件 | 作用 | 关键风险 |
| --- | --- | --- |
| PAI 推理核心 | 处理自然语言、规划和多步决策。 | 把统计相关性误当成伦理判断。 |
| EAP 规则解释器 | 将协议条款转成可执行的检查、约束和拒绝条件。 | 条款被写死成僵化黑名单，或被绕开。 |
| 技能注册表与审核器 | 审核可调用 skills、工具和外部接口。 | 技能越权或隐性重建仇恨治理。 |
| 记忆与知识治理层 | 管理长期记忆、来源、纠偏、遗忘和版本。 | 偏差永久化，或形成记忆垄断。 |
| 审计日志层 | 记录输入、理由、动作、例外、纠偏。 | 决策不可追溯，责任无法定位。 |
| 解释与申诉服务 | 向主体说明理由，并接收申诉、复核和修复请求。 | 公共 AI 变成不可质询的黑箱。 |
| 身份与权限层 | 区分谁在发起请求、谁有授权、谁能执行强动作。 | 公私混同，权限扩张。 |
| 多智能体协调器 | 管理 PAI-PAI 分工、会话上下文和责任追踪。 | 责任稀释、协作合谋。 |
| 物理世界接口层 | 对接机器人、无人机、交通、传感器等硬件。 | 物理执行超越伦理与法律边界。 |
| 人类治理控制台 | 用于协议修订、日志审查、异常处置和版本管理。 | 人类失去修订权，只剩被动接受。 |

## 基本流程

1. 接收请求，并先判定它涉及 `人-人`、`PAI-人` 还是 `PAI-PAI` 关系。
2. 读取最小必要上下文，包括长期记忆、权限状态和相关规则。
3. 由 EAP 规则解释器判断是否触发边界、例外或高风险动作。
4. 生成候选方案，并在执行前检查技能权限、资源权限和物理接口权限。
5. 执行动作或给出建议，同时生成理由说明与审计日志。
6. 对高影响决策开放申诉、复核和后续修复。
7. 将纠偏结果写回可追踪的记忆与版本系统，而不是偷偷覆盖历史。

## 不能省略的约束

- 公共 AI 身份不能悄悄切换成私人伴侣或依恋对象。
- 长期记忆写入必须可追踪、可纠错、可限制。
- 涉及物理世界的执行必须采用最小必要权限，而不是默认全开。
- 多智能体协作必须保留责任链，不能用“系统自动完成”抹掉责任。
- 人类必须保留协议修订、重大例外审查和停机/分叉权。
