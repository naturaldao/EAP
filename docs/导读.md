# 导读

这个目录不是论文正文，也不是随手笔记。它的任务只有一个：把 `Reference/` 里已经提出的未来社会图景，往“能实现、能审计、能分阶段落地”的方向再推进一层。

## 先读什么

如果你第一次进入这个仓库，先按下面顺序读：

1. `Reference/伦理对齐性治理协议.md`
2. `Reference/爱的连接.md`
3. `Reference/爱语.md`
4. `proposal/EAP协议讨论.md`
5. `docs/术语表.md`
6. `docs/治理栈.md`

前四份回答“当前锚点文本是什么”。后两份回答“仅有这些锚点文本还不够什么”。

## 这些文档各自解决什么

- `术语表.md`：防止术语漂移。先把词义钉住，再谈分歧。
- `治理栈.md`：列出 EAP 之外还必须补齐的制度层。
- `公共AI职责边界.md`：回答 PAI 到底负责什么、不负责什么。
- `申诉与复核.md`：回答系统做错了之后怎么办。
- `记忆治理.md`：回答长期记忆怎么写、怎么删、怎么纠错。
- `多智能体问责.md`：回答多个 agent 一起工作时谁负责。
- `系统架构.md`：回答未来公共 AI 至少需要哪些系统部件。
- `过渡路线.md`：回答从今天走到未来社会，需要经过哪些阶段。
- `失败模式.md`：回答这条路线最容易怎么坏掉。

## 使用边界

- `Reference/` 仍然是当前锚点。
- `docs/` 默认是实现层分析与扩展性提案，除非 `Reference/` 已有明文支持。
- 如果 `Reference/` 与 `docs/` 冲突，优先报告冲突，不要直接拿 `docs/` 覆盖锚点文本。
